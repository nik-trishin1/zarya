# UX/UI Analysis: Best Practices для zarya MVP

## 1. Анализ luma.com с точки зрения Product Manager & UX Designer

### Текущий путь пользователя на luma.com:
1. Пользователь открывает страницу
2. Видит "Discover Events" в навигации
3. Кликает на "Discover Events"
4. Попадает на страницу со списком событий
5. Кликает на событие
6. Видит детали события
7. Регистрируется

**Проблема:** 3 лишних клика перед тем, как увидеть события.

### Что luma.com делает хорошо:
- **Главная страница показывает события сразу** - "Popular Events" видны на главной
- **Карточки событий содержат обложку** - визуально привлекательно
- **Четкая иерархия информации** - дата, название, место
- **Одна кнопка для действия** - клик на карточку = детали события
- **Минималистичный дизайн** - не перегруженный

### Что можно улучшить для zarya:
- **Главная страница = список событий** (не отдельная страница "События")
- **Событие кликабельно целиком** (не нужна отдельная кнопка)
- **Регистрация видна на карточке** (или в деталях, но быстро)
- **Навигация минимальна** - только необходимые элементы

## 2. Best Practices для минимизации шагов до регистрации

### Принцип: Shortest Path to Value

Путь пользователя должен быть максимально коротким:
1. Открыть приложение → видит события
2. Клик на событие → видит детали
3. Клик "Зарегистрироваться" → готово

**Максимум 3 клика от открытия приложения до регистрации.**

### Навигация: Hamburger Menu или Bottom Tab Bar?

**Bottom Tab Bar (лучше для Telegram Mini App):**
- Всегда видна
- Быстрый доступ к главным функциям
- Привычна для мобильных пользователей
- Пример: Instagram, TikTok, Telegram

**Структура:**
```
[🏠 События] [🎫 Мои билеты]
```

Только 2 таба - максимум простоты.

### Главная страница (Home / События)

**Что показывать:**
1. Заголовок: "zarya + friends 🌅"
2. Список событий (все, в хронологическом порядке)
3. Каждое событие - карточка с:
   - Обложкой (cover image)
   - Датой и временем
   - Названием события
   - Местом
   - Статус регистрации (иконка ✅ или пусто)

**Что НЕ показывать:**
- Лишние кнопки
- Описание события (только в деталях)
- Список участников

### Карточка события

**Дизайн (вдохновлено luma.com):**
```
┌─────────────────────────┐
│  [Обложка события]      │  ← Cover image (новое в MVP!)
├─────────────────────────┤
│ Пт, 28 июня, 19:00      │  ← Дата и время
│ Вечеринка в доме        │  ← Название
│ Мой дом, Москва         │  ← Место
│                         │
│ [✅ Вы зарегистрированы]│  ← Статус (если зарегистрирован)
│ или                     │
│ [Зарегистрироваться]    │  ← Кнопка (если не зарегистрирован)
└─────────────────────────┘
```

**Важно:** Карточка целиком кликабельна → открывает детали события.

### Страница деталей события

**Путь:** Клик на карточку → модальное окно или новая страница с деталями

**Содержимое:**
1. Обложка (большая)
2. Название события
3. Дата, время, место
4. Описание (полное)
5. Количество зарегистрированных
6. Кнопка действия:
   - "Зарегистрироваться" (если не зарегистрирован)
   - "Отменить регистрацию" (если зарегистрирован)
7. Кнопка "Добавить в календарь" (если зарегистрирован)
8. Кнопка "Назад" или свайп вниз для закрытия

### Вкладка "Мои билеты" (Мои регистрации)

**Что показывать:**
- Список только зарегистрированных событий
- Те же карточки, что на главной
- Отсортированы по дате

**Иконка в навбаре:**
- 🎫 (билетик) или 📋 (список)
- Маленькая, не отвлекающая
- Показывает количество регистраций (бейдж с цифрой, если >0)

**Пример:** 🎫 (5) - означает 5 зарегистрированных событий

## 4. Путь пользователя (Optimized)

### Сценарий 1: Открыть приложение и зарегистрироваться

1. **Шаг 1:** Открыть приложение
   - Видит главную страницу со списком событий
   - Каждое событие - красивая карточка с обложкой

2. **Шаг 2:** Клик на карточку события
   - Открывается модальное окно или новая страница с деталями
   - Видит полное описание, дату, время, место

3. **Шаг 3:** Клик "Зарегистрироваться"
   - Подтверждение: "Вы зарегистрированы! ✅"
   - Видит кнопку "Добавить в календарь"

4. **Шаг 4 (опционально):** Клик "Добавить в календарь"
   - Скачивается .ics файл
   - Готово!

**Всего: 3-4 клика от открытия до регистрации.**

### Сценарий 2: Посмотреть свои регистрации

1. **Шаг 1:** Клик на 🎫 в навбаре
   - Видит список своих событий

2. **Шаг 2:** Клик на событие (если нужны детали)
   - Видит полные детали

**Всего: 1-2 клика.**

## 5. Event Cover Images - почему это важно в MVP

**Преимущества:**
- **Визуальная привлекательность** - события выглядят более реальными
- **Быстрое сканирование** - пользователь видит тип события по обложке
- **Брендирование** - обложки создают единый стиль zarya
- **Психология** - изображения привлекают внимание лучше, чем текст

**Как реализовать:**
- Admin загружает обложку при создании события
- Обложка хранится в S3 или Railway
- Отображается на карточке события (80x80px на списке, большая в деталях)

**Fallback:** Если обложки нет, показать цветной блок с иконкой события

## 6. Итоговые рекомендации для PRD

### Изменения в UX/UI:

1. **Главная страница = список событий** (убрать отдельную кнопку "События")
   - Пользователь открывает приложение и сразу видит события
   - Минус 1 клик

2. **Bottom Tab Bar вместо меню**
   - 🏠 События (главная)
   - 🎫 Мои билеты (мои регистрации)
   - Маленькие иконки в нижней части экрана

3. **Event Cover Images в MVP** (не на Iteration 2)
   - Admin может загружать обложку при создании события
   - Обложка отображается на карточке и в деталях

4. **Карточка события кликабельна целиком**
   - Не нужна отдельная кнопка "Подробнее"
   - Клик на карточку = открыть детали

5. **Регистрация видна на карточке и в деталях**
   - На карточке: иконка статуса или кнопка
   - В деталях: большая кнопка "Зарегистрироваться"

6. **Календарь после регистрации**
   - После успешной регистрации показать кнопку "Добавить в календарь"
   - Или в деталях события (если зарегистрирован)

### Убрать из MVP:
- Отдельная кнопка "События" (это главная страница)
- Лишние шаги в навигации
- Сложные фильтры и сортировки

### Добавить в MVP:
- Event Cover Images
- Bottom Tab Bar
- Статус регистрации на карточке события
